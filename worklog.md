# Android_Portfolio

* 2015년부터 2017년까지 제가 작업한 안드로이드 프로젝트들을 정리한 문서입니다.

## Ability

```text
    안드로이드를 주로 개발하며, 백엔드 서버 개발, 데이터베이스 관리 등도 필요에 따라 합니다.
    안드로이드는 주로 Java로 개발하며, 안드로이드에서 사용 가능한 새로운 언어인 Kotlin을 이용하여 개발할 때도 있습니다.
    CartaTagView (모서리가 각진 직사각형 모양의 버튼, 색상이나 코너값 등 커스터마이징 가능), CFView (카드뉴스 제작할 시 이용), DragListView (순서 이동이 필요한 리스트뷰), AliveFragmentView (커스텀 프래그먼트)등 필요에 따라 Custom View를 개발하여 사용합니다.
    오픈소스 라이브러리 사용을 지향합니다. 주로 MaterialDialog (머테리얼 다이얼로그에서 기본으로 지원되지 않는 기능들을 사용할 때 사용),  Material (머테리얼 관련 위젯들을 대부분 제공), SwipyRefreshLayout (아래로 스와이프해서 새로고침), BottomBar (Bottom에 메뉴가 존재하는 메뉴), Volley / Facebook Fresco (이미지 캐싱, 로드 라이브러리) 등을 사용합니다.
    서비스에 네이티브 로그인 외 SNS을 이용한 로그인을 지향합니다. Facebook (FacebookSDK), Twitter (Fabric), Naver (Naver API), Kakao (KakaoAPI), Github (Github SDK)와 연동한 바 있습니다.
    안드로이드에서 서버로 HTTP 요청을 보낼 때는 주로 Retrofit을 사용하며, 필요에 따라 LoopJ, HTTPUrlRequest을 사용하기도 합니다.
    NoSQL 데이터베이스를 지향합니다. 모바일에서는 Realm 데이터베이스, 서버에서는 MongoDB를 사용합니다.
    서버에서 안드로이드로 데이터를 보낼 때에는 미리 안드로이드에서 Token을 가져와 서버에 저장하고, 이를 이용해 알림을 보내는 Firebase Cloud Messaging을 사용합니다. (이전에는 Google Cloud Messaging을 사용하였습니다.)
    서버와의, 혹은 다른 클라이언트와의 실시간 연동이 필요할 때에는 Socket 통신을 이용합니다.
```

## 2015

> Bill.IM
* 2015 선린 모바일 콘텐츠 경진대회 은상

>> 역할

```text
    안드로이드 클라이언트 메인 프로그래머
    프로젝트 기획 보조
    Restful API 작성
```

>> 기획

```text

    필요할때 꼭 없는 그러한 여러 물건들을 같은 단체 내에서 빌릴 수 있도록 해주는 서비스. 중고 제품의 물물교환도 가능하도록 하여, 단체 내에서 새로운 물건을 구매 하는 것이 아닌 서로 필요한 것을 주고받으며 도울 수 있도록 만들어 주는 서비스.
    임대 기능, 신고 기능, 중고 제품 교환 기능 제공.
```

>> 기술

```text
    서버는 Node.JS로 구동하였으며 CoffeeScript로 작성되었고, MongoDB를 사용하였음.
    클라이언트는 안드로이드로 구현하였으며, Realm 데이터베이스와 SharedPreferences를 사용하였음.
    FloatingActionButton, Material, MaterialViewPager, MaterialEditText, SwipyRefreshLayout, MaterialDialog 라이브러리를 사용하였음.
    서버 -> 안드로이드 클라이언트에는 Google Cloud Messaging을 사용하였음.
    안드로이드 클라이언트 -> 서버에는 HTTP 통신을 이용하였음. (Retrofit 사용)
    각각의 거래 항목은 데이터베이스에 저장하였으며 변경사항이 요청자 / 거래자로부터 발생하면 GCM을 이용해 클라이언트에 푸시를 보내고, 변경사항을 데이터베이스에 업데이트하였음.
```

> Knock
* 2015 선린 디지털 콘텐츠 경진대회 동상

>> 역할

```text
    안드로이드 클라이언트 메인 프로그래머
    Restful API 작성
```

>> 기획

```text
    보안이 중요시되는 현대 사회에서 공동으로 사용하는 공간에서 보안에 취약한 컴퓨터들을 위해서 카드 태깅을 이용하여 PC Lock / Unlock할 수 있는 플랫폼.
    PC에서는 로그인, 로그아웃, 카드를 이용한 잠금 / 잠금 해제를 제공함.
    모바일에서는 로그인, 로그아웃, 카드를 이용하지 않은 잠금 / 잠금 해제를 지원함.
```

>> 기술

```text
    서버는 Node.JS로 구동하였으며 JavaScript (ECMAScript 5.1)로 작성되었고, MongoDB를 사용하였음.
    PC 클라이언트는 DirectX API를 이용하여 창을 구현하였음.
    모바일 클라이언트는 안드로이드로 구현하였으며, SharedPreferences를 사용하였음.
    PC <-> 서버, 서버 <-> 안드로이드간의 실시간 통신에는 Socket을 사용하였음.
    MaterialDialog 라이브러리를 사용하였음.
    안드로이드 클라이언트 -> 서버에는 HTTP 통신을 이용하였음. (Retrofit 사용)
    Lock / Unlock 요청이 들어오면 서버의 데이터베이스에 현재 상태를 저장하고, Socket을 이용하여 요청 상황을 각 클라이언트에 브로드캐스트시키는 방식을 사용하였음.
```

```text
    바코드 스캔을 통해 유통기한을 확인하여 화장품 관리를 할 수 있고, 현재 구매된 화장품 데이터를 축적하여 가장 인기있는 화장품 랭킹을 측정할 수 있다.
    보관, 알림, 랭킹, 추천 기능을 제공함.
```

>> 기술

```text
    서버는 Python로 구동하였으며 SQL을 사용하였음.
    클라이언트는 안드로이드로 구현하였으며, 계정 정보 저장, 화장품 정보 캐싱 위해 Realm 데이터베이스를 사용하였으며 설정값 저장을 위해 SharedPreferences를 사용하였음.
    RecyclerView를 사용하는 데 있어서 효율적 관리와 간결한 코드를 구성하기 위해 LastAdapter 라이브러리를 사용하였음.
    안드로이드 클라이언트 -> 서버에는 HTTP 통신을 이용하였음. (Retrofit 사용)
    MaterialDialog, FAB, BottomBar 라이브러리를 사용하였음.
```

>> 기획

```text
    비트코인을 거래할 수 있는 모든 거래소의 거래 현황을 보여주고, 자신의 거래 포트폴리오를 관리할 수 있음.
    코인 정보가 일정 이상 / 이하일 시 알림을 보내줌.
```

>> 기술

```text
    서버는 Node.JS (JavaScript(ECMAScript 6))와 Python로 구동하였으며 MongoDB를 사용하였음.
    클라이언트는 안드로이드로 구현하였으며, 계정 정보 저장, 코인 데이터 캐싱 위해 Realm 데이터베이스를 사용하였으며 설정값 저장을 위해 SharedPreferences를 사용하였음.
    RecyclerView를 사용하는 데 있어서 효율적 관리와 간결한 코드를 구성하기 위해 LastAdapter 라이브러리를 사용하였음.
    순서 편집이 가능한 리스트 구현을 위해 DragListView 커스텀뷰를 구현하여 사용함.
    안드로이드 클라이언트 -> 서버에는 HTTP 통신을 이용하였음. (Retrofit 사용)
    서버 -> 클라이언트에는 Firebase Cloud Messaging을 사용하였음.
    MaterialDialog, FAB 라이브러리를 사용하였음.
```

>> 기획

```text
    요즘 페이스북 등 SNS에서 공유되는 카드뉴스들을 한 곳에서 모아서 보고, 공유하고, 토론할 수 있는 카드뉴스 SNS 플랫폼.
    카드뉴스 태그에 따른 분류, 검색, 좋아요, 댓글, 토론이 가능함.

```

```text
    서버는 Node.JS (JavaScript(ECMAScript 6))로 구동하였으며 MongoDB를 사용하였음.
    클라이언트는 안드로이드로 구현하였으며, 계정 정보 저장을 위해 Realm 데이터베이스를 사용하였으며 설정값 저장을 위해 SharedPreferences를 사용하였음.
    RecyclerView를 사용하는 데 있어서 효율적 관리와 간결한 코드를 구성하기 위해 LastAdapter 라이브러리를 사용하였음.
    안드로이드 클라이언트 -> 서버에는 HTTP 통신을 이용하였음. (Retrofit 사용)
    카드뉴스를 모바일에서 직접 제작할 수 있게 CFView를 제작함. (카드뉴스 편집기)
    서버 -> 클라이언트에는 Firebase Cloud Messaging을 사용하였음.
    CartaTagView, MaterialDialog, FAB 라이브러리를 사용하였음.
```
